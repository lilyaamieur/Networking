(e) Why could it be that some interfaces do not have a MAC address?

1. Virtual Interfaces (e.g., Loopback Address)

  • The loopback interface with the IP address 127.0.0.1 (or ::1 in IPv6) does not need a MAC address because it only performs communication within the host.
  • Loopback interfaces do not interact with the physical layer; they only operate at higher layers (transport and network layers).

2. Virtual Machines and Containers

  • Virtual machines (VMs) and containers may use virtual network interfaces that initially lack a MAC address.
  • However, many hypervisors assign virtual MAC addresses to these interfaces for network emulation.
    • For example, KVM, VMware, and Docker often generate virtual MACs to allow communication with external networks.

3. Key Networking Concepts:

Round Trip Time (RTT):

  • The time it takes for a packet to travel from the source to the destination and back.

Time To Live (TTL):

  • TTL is a value that limits the lifespan of a packet in the network.
  • In the ping results, the TTL was 58, meaning the packet could pass through 58 routers before being discarded if no response is received.

Packet Loss:

  • Packet loss occurs when data fails to reach its destination. In the test, there was 0% packet loss, indicating a stable connection.

1. Load Balancing with Multiple IPs

  • Websites often have multiple IP addresses to support load balancing.
  • DNS-based load balancing may change the IP address returned for the same domain at different times.
  • Example: A server can respond for one hour, and another server may take over in the next hour to distribute the load.

2. DNS Resolution Process

When accessing a website (e.g., https://www.google.com), the computer needs the IP address. The lookup process is as follows:

  1. Browser Cache: Checks if the IP address is already cached.
  1. OS Cache: If not in the browser, checks the operating system's DNS cache.
  1. Home Router DNS: If not in the OS cache, queries the router's DNS server.
  1. ISP DNS Server: If not found, the router forwards the request to the ISP's DNS server.

There are 13 root DNS servers spread globally, responsible for the top-level DNS hierarchy.

3. Recursive DNS Resolution Example

  • During recursive DNS resolution, a query is sent from one DNS server to another until the IP address of the requested website is found.

NOTES
  • IP Address (Internet Protocol Address):
    • Used to identify a device on a network and enable routing between devices across different networks.
    • Think of it like the postal address of a device, which is used to locate and deliver data across the internet or a local network.
    • Example format:
      • 192.168.1.1 (IPv4)
      • 2001:0db8:85a3:0000:0000:8a2e:0370:7334 (IPv6)
    • IP Address Format:
      • IPv4: 32-bit address, written as four decimal numbers separated by periods (e.g., 192.168.1.1).
      • IPv6: 128-bit address, written as hexadecimal numbers separated by colons (e.g., 2001:0db8::1).
    • Scope
      • Identifies a device on a global scale (internet-wide or across different networks).
    • Example Usage
      • Used by routers and devices to send data across the internet. For example, your IP address is used when you visit a website.
  • MAC Address (Media Access Control Address)
    • Used to uniquely identify a device’s network interface at the hardware level.
    • It operates at the data link layer (Layer 2) of the OSI model and ensures that data is delivered to the correct device on the same local network.
    • Example format:
      • 00:1A:2B:3C:4D:5E
    • MAC Address Format:
      • 48-bit address, written as six pairs of hexadecimal digits, usually separated by colons or hyphens (e.g., 00:1A:2B:3C:4D:5E).
    • Scope
      • Identifies a device only on the local network (within the same subnet or local area network).
    • Example Usage:
      • Used by switches and local routers to ensure the data packet is sent to the correct device within the same local network.

    DHCP Network:

    • DHCP (Dynamic Host Configuration Protocol) is used to dynamically assign IP addresses to devices in a network.

    OSI Model (7 Layers):

    The OSI model is a conceptual framework used to describe network communication in 7 layers:

    1. Application Layer
    1. Presentation Layer
    1. Session Layer
    1. Transport Layer
    1. Network Layer – IP addresses are used here.
    1. Data Link Layer – MAC addresses are used here.
    1. Physical Layer – Represents the physical network card (e.g., Wi-Fi card).

    Loopback Interface:

    • The loopback interface (usually 127.0.0.1) does not interact with physical network cards.
    • The loopback interface allows a computer to communicate with itself.
    • Question: Why does the loopback have a MAC address?
      • Even though the loopback interface does not reach the physical network card, it can still be assigned a MAC address for internal operations within the system. However, it does not rely on hardware communication.

    TCP/IP Stack:

    • The TCP/IP Stack is a more practical model for networking.
    • Key Concepts:
      • Network Layer: Deals with IP addresses.
      • Data Link Layer: Deals with MAC addresses.
      • Physical Layer: Manages the hardware, such as the Wi-Fi card.

    Network Tools:

    • Ping: Used to test network connectivity.
      • Sends packets and measures the Round Trip Time (RTT), the time it takes for a packet to go to the destination and return.
      • TTL (Time to Live): A countdown in milliseconds (e.g., 60ms). Each time a packet reaches a node, the TTL is reduced. If TTL reaches zero, the packet is destroyed, meaning the packet was lost in the network.

    Timeouts and Firewalls:

    • A timeout occurs when no response is received after sending a request. This could indicate network issues or intentional blocking by a firewall.
    • Some servers are configured not to respond to ping requests for security reasons, even though they are functional when accessed through a browser. This helps in preventing certain types of network attacks (like ping floods).
