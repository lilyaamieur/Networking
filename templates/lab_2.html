What’s the difference between standard cables?

Standard cables typically have 8 ports. In these cables, there is a solid brown pair, and a twisted pair consisting of white and brown. The green, orange, and blue pairs are used for transmitting and receiving signals to minimize noise. Specifically, the blue pair is used for transmission, while the orange pair is used for reception.

Power over Ethernet (PoE) allows you to supply power to devices without needing a separate power plug.

What happens if we connect the cable to two computers?

Connecting the cable in this manner is akin to having two people trying to communicate but connecting their ears to each other and their mouths to each other—resulting in both parties attempting to talk and listen simultaneously (talk-talk // listen-listen).

In this setup, both devices are configured to transmit rather than receive, leading to communication failure. To facilitate proper communication, we need to use a different type of cable where the transmission and reception pairs are swapped. This configuration is called a

cross cable, which allows the recipient to receive while the transmitter sends signals.

What is the subnet mask?

The subnet mask allows you to determine which part of the IP address represents the network ID and which part represents the host ID.

For example, in the IP address 192.168.1.2:

  • The subnet mask 255.255.255.0 indicates that the first three octets (192.168.1) represent the network ID, while the last octet (2) represents the host ID.

Broadcast Address:

The address 192.168.1.255 is used as the broadcast address. This address allows communication to all devices on the same subnet.

    Why do we not assign an IP address to the switch?

    Typically, a switch operates at Layer 2 of the OSI model (the Data Link layer) and is responsible for forwarding data packets based on MAC addresses rather than IP addresses. As such, it doesn't need an IP address for standard operation in a local area network (LAN). However, if the switch is a managed switch, it may have an IP address for management purposes, allowing you to configure and monitor it remotely.

    What’s the difference between a hub and a switch?

    • Hub: A hub is a basic networking device that connects multiple computers in a network. It broadcasts data to all devices connected to it, regardless of the intended recipient. This means that every device sees all the traffic, which can lead to collisions and inefficiencies.
    • Switch: A switch is more advanced than a hub. It operates by intelligently forwarding data only to the specific device (or port) that is intended to receive it, using MAC addresses to identify devices. This greatly reduces network congestion and increases overall efficiency.     Visualizing the Switch:

    You can think of a switch as a straight cable that allows you to connect many computers. It creates a point-to-point connection between the devices, enabling them to communicate effectively.

    Connecting Networks:

    While switches allow for efficient communication within a local network, they do not connect different networks. For that purpose, you need a router. A router enables communication between your local network and other networks (such as the internet) by routing data packets between them.
